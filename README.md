# 🧪 Conduit QA Testing Project

## 📌 Overview
This project focuses on manual testing of the authentication features (Sign In and Sign Up) of the Conduit web application.

The project was developed as part of a QA training program to apply real-world testing practices using industry tools.

---

## 🎯 Objective
- Design and execute test cases for user authentication features
- Validate input fields and business rules
- Ensure proper error handling and user feedback

---

## 🔍 Scope
### Covered Features:
- Sign In (Login)
- Sign Up (Registration)

### Test Types:
- Positive testing
- Negative testing
- Boundary value analysis
- Validation testing

---

## 🛠️ Tools Used
- TestRail (Test Case Management)
- Jira (Test Management & Traceability)

---

## 🧪 Test Design
Test cases were created based on functional requirements, covering:

- Username validation (length, format, uniqueness)
- Email validation (format and uniqueness)
- Password rules (length, complexity, constraints)
- Confirm password matching
- Required field validation
- Navigation (Sign In / Sign Up flow)

---

## ▶️ Test Execution
- Test runs were created and executed in TestRail
- Each test case includes:
  - Execution status (Passed/Failed)
  - Expected vs Actual results
  - Comments for traceability

---

## ⚠️ Observations
During the process, partial integration between Jira and TestRail was identified.  
Full linking was not possible due to the environment configuration limitations.

---

## 📁 Project Structure
- Test cases organized by feature and field:
  - Sign Up Form
  - Username Field
  - Email Field
  - Password Field
  - Confirm Password Field
  - Sign Up Button
  - Navigation Link
